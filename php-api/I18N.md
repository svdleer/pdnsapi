# API Documentation Internationalization (i18n)

This API supports bilingual documentation with English (default) and Dutch translations.

## Available Languages

- **English** (`en`) - Default language
- **Dutch** (`nl`) - Nederlandse vertaling

## Access URLs

### Documentation Interface
- **English**: `/docs` or `/docs?lang=en`
- **Dutch**: `/docs?lang=nl`

### OpenAPI Specifications
- **English**: `/openapi` or `/openapi.yaml`
- **Dutch**: `/openapi-i18n.php?lang=nl`

## Implementation Details

### Files
- `translations.php` - Translation strings for both languages
- `openapi-i18n.php` - Dynamic OpenAPI generator with i18n support  
- `docs.html` - Enhanced documentation interface with language switcher
- `index.php` - Updated routing to support localized OpenAPI

### Features
- **Automatic fallback** - Invalid language codes default to English
- **Dynamic generation** - OpenAPI spec generated on-the-fly with translations
- **Comprehensive translation** - All main descriptions, endpoints, and error messages
- **Language switcher** - Easy switching between languages in the documentation interface

### Translation Coverage
- API title and description
- All endpoint summaries and descriptions
- Tag descriptions
- Common response messages
- Error messages

## Adding New Languages

To add a new language (e.g., German 'de'):

1. **Add translations to `translations.php`**:
```php
'de' => [
    'api.title' => 'PDNSAdmin PHP API',
    'accounts.list.summary' => 'Alle Konten auflisten',
    // ... more translations
]
```

2. **Update language switcher in `docs.html`**:
```html
<a href="?lang=de">🇩🇪 Deutsch</a>
```

3. **Test the new language**:
```bash
curl http://localhost/php-api/openapi-i18n.php?lang=de
```

## Usage Examples

### Curl Examples
```bash
# Get English OpenAPI spec
curl http://localhost/php-api/openapi

# Get Dutch OpenAPI spec  
curl http://localhost/php-api/openapi-i18n.php?lang=nl

# View Dutch documentation interface
curl http://localhost/php-api/docs?lang=nl
```

### Browser Access
- Visit `/docs` for English documentation
- Visit `/docs?lang=nl` for Dutch documentation
- Use the language switcher flags in the navigation

## Technical Notes

- Language detection via `$_GET['lang']` parameter
- Fallback to English for invalid/missing language codes
- YAML output with proper `Content-Type: application/x-yaml` headers
- Translation function prevents key lookup errors
- Swagger UI automatically updates based on selected language
